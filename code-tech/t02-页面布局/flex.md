# 课 2 flex 布局：空间的魔术师

## 为什么需要 flex

各位同学是否遇到过这样一个问题：排列元素的时候，我们只关心元素是否排列整齐，间隔是否平均，并不想去计算元素之间具体是多少大小——毕竟换一个屏幕它可能就变了。

在之前的课上我们曾讲过，在制作网页的时候，我们首先要对页面进行拆解。比如说我们先将整个页面拆为 header、main、footer 等部分，再将 header 拆成 logo、navigation、search 等，一步步拆解到元素不能再拆解为止。

假如 header 拆解后得到的是 logo、navigation（里面有三个导航按钮）、search。我们希望 logo、navigation、search 在整个 header 里所占的比例为 1：3：1，多余的空间平均分配在它们两边和中间。

如果单纯利用盒子模型，先不提不同屏幕像素不同的问题，光是需要计算出每个元素的大小、再计算出每个间隔的大小，就让我们很头疼，而这只是一个仅有 5 个元素的布局。

虽然麻烦，但是并不是完全没法做，硬着头皮尚可一战。但是突然我们的需求改了：navigation 里面的导航按钮数量不确定，可能是 3 个，也可能是 5 个。logo、navigation、search 比例仍然为 1：3：1。虽然导航按钮的个数不确定，但是不管是几个，我们都要求它在 navigation 里平均分布（假设 navigation 空间足够大）。

至此，使用盒子模型的同学口吐芬芳：难不成我要给每个数量单独写一个 html？（这样做只会被领导口吐芬芳）

## flex 的作用

首先我们需要提一下 flex 布局是怎么使用的。flex 布局由两个部分组成：父元素：flex 容器，和子元素。在父元素的 css 中加入`display: flex`就能开启 flex 布局，开始支配子元素。

我们先从子元素开始讲起。子元素最关键的是 3 个属性：flex-grow flex-shrike flex-basis。flex-grow 描述了元素是怎么争夺多余空间的，默认是 0，表示不会把多余的空间占为己有。

与普通的利用盒子模型布局的方法不同，对于父元素来说，它只在乎空间是怎么分配的，并不在乎有几个子元素。空间可以是分配在每个子元素之间，但并不在乎每个子元素分配到了
